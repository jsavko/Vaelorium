# v0.2.0 — Sync

**Date:** 2026-04-13

## Headline

**Sync is here.** Back up every edit to your own folder, S3 bucket, or any S3-compatible service (AWS, Cloudflare R2, Backblaze B2, Wasabi, Minio, Garage). Work across as many devices as you want — all end-to-end encrypted with a passphrase only you know.

## What's new

### Sync foundation (M7 complete)
- **End-to-end encrypted** with Argon2id-derived keys and ChaCha20-Poly1305 AEAD.
- **Pluggable backends** — filesystem folder or any S3-compatible bucket via `aws-sdk-s3`.
- **Per-Tome opt-in** — some Tomes sync, others stay purely local.
- **Hybrid journal + snapshot protocol** — fast for active Tomes, compact on disk (~97% reduction on typical Tomes thanks to gzip before encryption).
- **12 of 14 user-data tables sync** — pages (including full content), entity types + fields + values, relations, tags, maps + pins, timelines + events, boards + cards + connectors, relation types. Images + page version history + wiki links are intentionally local-only.

### Multi-device UX
- **First-run setup wizard** walks through backend choice, passphrase, device name.
- **Restore from backup** on any fresh device — pick a Tome from the picker's restore panel and Vaelorium downloads + decrypts the latest snapshot automatically.
- **Conflict resolver modal** lists every conflicted field across every synced table with per-row Local/Remote pickers and bulk resolve.
- **Device name stub** — two devices named "My Laptop" stay distinguishable (auto-appended 4-hex suffix).
- **Activity viewer** — Settings → Sync → "Recent activity" shows the last 100 sync ticks with timing, op counts, conflicts, errors.
- **OS keychain integration** — passphrase persists between launches on macOS, Windows, and Linux (with a working Secret Service backend).
- **Exponential-backoff retry** on transient backend errors (1 s / 4 s / 16 s) before surfacing failure.

### Other improvements
- Path-independent Tome identity — bucket prefix derives from a stable per-Tome UUID, so the same Tome produces the same backend location on every device regardless of where the file lives locally.
- Conflicts cleared automatically on the second device when the first device ships its resolution.
- Tome identity stored in `tome_metadata`; lazy-created on first sync.

## Breaking changes

- **Bucket layout changed** from path-hashed prefixes to UUID-hashed prefixes (5.0). If you were testing a pre-0.2.0 sync branch, delete the old `tomes/<hash>/` entries from your bucket and re-enable sync — fresh prefix under `tomes/<uuid>/`. Clean re-setup precedent from the app-global backup refactor.
- **Backup configuration schema changed.** Migration 010 drops per-Tome backend config; migration 011 moves device_name to app-global; migration 012 adds the sync activity log. All automatic on first launch.

## Known limitations

- **Images don't sync yet.** Media blobs are local-only; M8 will add a separate file-sync layer.
- **Lost passphrase is unrecoverable.** This is the standard E2EE posture. Write it down.
- **Multi-Tome background sync.** Only the currently-open Tome syncs in the background for now. Multi-Tome poll needs a per-Tome pool registry — planned fast-follow.
- **Web view / read-only reader** — deferred to a later minor.
