# Milestone 2.5: Tomes (Multi-Project Support)

**Date:** 2026-04-11

## Summary

Each campaign/world is now a self-contained `.vaelorium` file. The app starts with a Tome Picker home screen. Users can create new Tomes, open existing ones, and switch between them. The Rust backend supports hot-swapping database connections.

## Changes

### Backend
- Refactored `DbPool` to `ManagedDb` (`Arc<RwLock<Option<DbPool>>>`) — app starts with no DB
- All existing commands updated to handle no-Tome state (returns error if no Tome open)
- New `app_state.rs` for reading/writing recent Tomes list (JSON file in app data dir)
- New `tomes.rs` commands: create_tome, open_tome, close_tome, get/update_tome_metadata
- New migration `003_tome_metadata.sql` — key-value metadata table per Tome
- `open_database()` function for creating/connecting to SQLite at arbitrary path

### Frontend
- `TomePicker.svelte` — home screen with recent Tomes grid, Create New card, Open File button
- `CreateTomeModal.svelte` — name + description form for new Tomes
- `tomeStore.ts` — reactive state for current Tome, recent Tomes, open/close actions
- `tomes.ts` API wrappers for all Tome commands
- Bridge mock supports Tome commands (auto-open in browser, resetable DB)
- App.svelte switches between Tome Picker and editor based on `isTomeOpen`
- Sidebar shows Tome name in header, accepts `onCloseTome` prop

### Tests
- 3 new E2E tests for Tome system
- All 80 E2E + 44 unit = 124 tests passing
