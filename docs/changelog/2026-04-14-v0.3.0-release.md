# v0.3.0 — Vaelorium Cloud

**Date:** 2026-04-14

## Headline

**Vaelorium Cloud** — a hosted, end-to-end encrypted backup + sync tier for users who don't want to configure their own S3 bucket or manage a folder sync agent. Sign in with your Vaelorium account, pick an encryption passphrase, and every edit is backed up and synced across every device you sign in on.

Filesystem + S3 backends from v0.2.0 still work unchanged — Cloud is a new option, not a replacement.

## What's new

### Vaelorium Cloud (hosted tier)
- **Zero-knowledge** — your data is encrypted on-device before upload with the existing M7 per-Tome passphrase crypto (Argon2id + ChaCha20-Poly1305). Vaelorium Cloud never sees your content.
- **Signup via web only** — sign up at [cloud.vaelorium.com/signup](https://cloud.vaelorium.com/signup); the app signs in, it doesn't sell subscriptions.
- **Two-password model:** your Vaelorium **account password** authenticates to the service (dual-KDF Argon2id → auth_hash proves you know it without revealing it); your **encryption passphrase** is separate and encrypts your data. The account password can be reset (M6+ recovery-phrase flow); the encryption passphrase cannot — write it down.
- **OS keychain** stores the device token and a wrapped account_key so you don't re-sign-in every launch.
- **Backup setup wizard** lists Cloud first with a `recommended` pill. Filesystem and S3 remain available for BYO-cloud setups.
- **"Sign in needed" pill state** — if your device token is revoked from the web dashboard, the sidebar pill turns red and routes to the Backup tab for re-signin.

### Under the hood
- New `BackendKind::Hosted` alongside `Filesystem` and `S3`. Engine + registry + conflict detection + activity log + retry/backoff + restore-from-backup all run polymorphically through the trait — no duplicate code paths per backend.
- 3 new `BackendError` variants (`Unauthorized`, `PaymentRequired`, `QuotaExceeded`) — all hard-no-retry in the runner. `Unauthorized` routes to the distinct "unauthorized" sync status so the UI can prompt re-signin.
- Tauri commands `cloud_signin`, `cloud_signout`, `cloud_status`; multi-entry keychain under the `vaelorium-cloud` service (5 named entries, isolated from the existing backup-passphrase entry).
- Cloud URL configurable via `VAELORIUM_CLOUD_URL` env var for staging / local dev; defaults to `https://cloud.vaelorium.com`.

## Breaking changes

None. Existing Filesystem + S3 backup configurations continue working unchanged. The new `BackendKind::Hosted` is additive.

## Under the hood (tests)

- 84 library unit tests + 13 sync integration + 10 hosted_backend wiremock + 4 hosted_auth wiremock passing.
- Argon2id dual-KDF pinned to m=65536, t=3, p=1, len=32, v=0x13 (matches cloud server exactly); XChaCha20-Poly1305 (24-byte nonce) for account_key wrap.

## Known limitations / deferred

- **Recovery-phrase flows** (password reset without data loss) — wire format is frozen, UI lands in a later minor.
- **Multipart upload** for >100 MB blobs — not required for v1.
- **Mobile** — desktop-only still.
- **In-app signup** — stays web-only (Stripe Checkout); signing up via the app would require embedding a payment flow, which we're not doing.

## Upgrade notes

Open the app. Settings → Backup → "Set up backup…" → pick Vaelorium Cloud → sign in with your cloud account → pick an encryption passphrase. If you were already using Filesystem or S3, nothing changes — don't touch your existing backup.
